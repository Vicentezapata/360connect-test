Sistema de Tour Virtual 360° con Portal de Administración para sucursales — despliegue on-premise en Docker Compose.

📅 2026-05-29
🏗️ Stack: React 19 · NestJS · PostgreSQL · MinIO
📦 Despliegue: Docker Compose (on-premise)
🖥️ Interfaces: 33 (9 Viewer + 24 Admin)
⏱️ Estimación: 5–7 meses
PRD› Arquitectura› Interfaces› Historias› Plan› Roadmap
PRD Epic — Plataforma de Tour Virtual
Descripción del problema, solución y alcance del epic completo.
🎯

Goal: Desarrollar desde cero una plataforma web moderna con viewer 360° + portal admin + despliegue on-premise en Docker Compose, instalable en el servidor de cada cliente en < 30 minutos.

El Problema

La organización gestiona ~20 sucursales con un software legado que no permite administrar contenido de forma autónoma, carece de planos interactivos, no tiene arquitectura web moderna y no puede instalarse como instancia independiente por cliente. Los técnicos de campo no pueden acceder a información de infraestructura con baja conectividad.

La Solución

🌐
Viewer Público 360°
Tour virtual con menú contextual, paneles de infraestructura, formulario de reporte de campo y soporte offline PWA.
9 pantallas
🛠️
Portal de Administración
CRUD completo, editor de hotspots 360°, editor de planimetría en canvas y biblioteca de documentos.
24 pantallas
🐳
Despliegue On-Premise
Docker Compose + script de instalación automatizado. Setup completo en < 30 minutos.
Zero cloud deps
📱
PWA Offline
Viewer y Admin funcionan sin conexión. Cola de escritura en IndexedDB, sincronización automática al reconectar.
IndexedDB + SW
01 Personas de Usuario
Cuatro roles con necesidades y contextos distintos.
👷
U1
Técnico de Campo

Visita sucursales físicamente. Necesita info de infraestructura. Puede tener baja/nula conectividad. Usa Chrome o Firefox.

🎛️
U2
Admin de Contenido

Gestiona fotos 360°, planos, documentos e informes desde el portal. Necesita soporte offline para trabajo en campo.

⚙️
U3
Super Administrador

Instala la instancia, gestiona usuarios y roles, configura el branding del cliente. Responsable de backups.

📊
U4
Gerente / Supervisor

Consulta informes y documentación de forma ocasional. Solo lectura, sin capacidad de subir contenido.

02 User Journeys de Alto Nivel
Cuatro flujos principales que guían el diseño del sistema.
🗺️ J1 — Ver Tour Virtual de una Sucursal
1Técnico accede a la URL del viewer → pantalla de login (V-09): ingresa credenciales
2Autenticado → lista de sucursales con portada y estado
3Selecciona sucursal → carga visor 360° del primer espacio
4Navega entre espacios usando hotspots en la imagen
5Abre menú contextual → selecciona [COMUNICACIONES]
6Panel lateral muestra tabla de enlaces, proveedor, estado
7Descarga documento técnico adjunto en PDF
🛠️ J2 — Administrar una Sucursal
1Admin entra al portal → dashboard con resumen
2Sección Sucursales → sube foto 360° (drag & drop)
3Sistema valida, genera thumbnail, muestra progreso
4Pestaña "Planimetría" → abre editor canvas
5Sube imagen base del plano → posiciona marcadores
6Guarda versión con comentario → publica planimetría
📵 J3 — Trabajar Offline en Campo
1Admin abre el portal (previamente cargado) → sin conexión: banner "Modo offline"
2Edita datos de infraestructura de una sucursal
3Los cambios se guardan en IndexedDB
4Cuando vuelve la conexión: banner "Sincronizando..."
5Cambios se envían a la API automáticamente → confirmación visual
🚀 J4 — Instalar Nueva Instancia
1Técnico ejecuta: curl -fsSL https://install.ejemplo.com | bash
2Wizard de setup: nombre empresa, logo, colores, dominio
3Se configura Docker Compose, SSL automático (Let's Encrypt)
4Se migran fotos 360° existentes desde ruta local
5Sistema listo en < 30 minutos ✓

05 Arquitectura del Sistema
Vista general del stack on-premise en Docker Compose, servicios y comunicaciones.
🏗️

Todo el sistema corre en el servidor del cliente en un único Docker Compose stack. No hay dependencias de servicios cloud externos. El almacenamiento de archivos usa MinIO (API S3-compatible self-hosted). Las imágenes 360° solo son accesibles mediante presigned URLs con expiración.
